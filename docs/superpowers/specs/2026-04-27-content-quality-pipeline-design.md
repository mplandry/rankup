# Content Quality Pipeline — Design Spec
**Date:** 2026-04-27
**Status:** Approved

## Overview

RankUp is in soft launch with a small group of testers. Questions are generated by AI (Claude and ChatGPT). The primary concern before scaling up is content quality: originality/copyright risk, weak distractors, and imbalanced answer distribution. No questions have been uploaded yet.

This spec covers four connected improvements, all in the admin panel, that form an end-to-end content quality pipeline:

```
Import CSV → Auto-flag issues → Review queue → Approve → Serve to students
```

Nothing reaches students unless it has `review_status = 'approved'`.

---

## 1. Database Changes

One migration adding six columns to the `questions` table:

```sql
review_status           text         NOT NULL DEFAULT 'pending'
  -- 'pending' | 'approved' | 'needs_revision'

review_notes            text
  -- admin-written note when marking needs_revision

distractor_score        integer
  -- 0–100 quality score from Claude API; null if not yet evaluated

distractor_notes        text
  -- AI-generated explanation of the weakest distractor (set on import, read-only to admin)

originality_reviewed    boolean      NOT NULL DEFAULT false
  -- admin manually confirms question is not verbatim from source

originality_reviewed_by text
  -- user_id of the admin who checked originality

originality_reviewed_at timestamptz
  -- timestamp of originality check
```

**Behavior change:** `/api/sessions` currently filters `is_active = true`. It must also filter `review_status = 'approved'` so pending and flagged questions never reach students.

All imported questions start as `pending`. Existing questions (once uploaded) also default to `pending` until reviewed.

---

## 2. Review Queue

A dedicated admin page at `/admin/review` that surfaces questions needing attention.

### Layout
- **Left panel:** scrollable queue list showing pending and needs_revision questions. Each item shows status badge, distractor score (if available), and the first line of question text.
- **Right panel:** full question detail for the selected item.

### Question Detail Panel
1. **Source reference banner** (prominent, dark navy) — book title, edition, chapter name, page range, topic. Lets the admin open the physical book to verify originality.
2. **Distractor quality alert** (yellow warning) — shown when `distractor_score < 60`. Displays the AI's specific explanation of which option is weak and why.
3. **Question card** — full question text with all four answer options. Correct answer highlighted in green; flagged distractor highlighted in red.
4. **Originality toggle** — checkbox labeled "Confirm this question is not verbatim from pp. X–Y." Checking it sets `originality_reviewed = true`, records `originality_reviewed_by` and `originality_reviewed_at`.
5. **Review notes field** — freetext, required when marking Needs Revision.
6. **Action buttons:** Needs Revision · Skip · Approve

### Queue Behavior
- Approve → sets `review_status = 'approved'`, advances to next question
- Needs Revision → sets `review_status = 'needs_revision'`, stores notes, advances
- Skip → leaves status unchanged, advances
- Queue header shows live counts: "14 pending · 2 flagged"

## 3. Distractor Quality Check

When questions are imported, each is sent to the Claude API for distractor evaluation.

**Prompt intent:** Given the question text and four answer options (with the correct answer identified), score the quality of the wrong answers on a 0–100 scale. Deduct heavily for: absolute language ("always"/"never"), options that are obviously off-topic, options that are trivially shorter or longer than the others, and options that repeat the question stem. Return a score and a one-sentence explanation of the weakest distractor.

**Result storage:** `distractor_score` (integer) and a brief explanation stored in `distractor_notes` (text). These are set on import and displayed read-only in the review queue. Admin notes go separately into `review_notes`.

**Threshold:** Questions with `distractor_score < 60` are surfaced at the top of the review queue and shown with the yellow warning banner.

**API cost:** Run once per question on import. Not re-run unless admin triggers it manually. This keeps costs predictable.

## 4. Answer Distribution Panel

Admin panel at `/admin/questions` (or a tab on the admin dashboard) showing correct-answer distribution stats.

### Overall View
- Bar chart: percentage of approved questions with correct answer A / B / C / D
- Target range: 20–30% per letter
- Plain-English advisory when any letter is outside the target range (e.g., "C is over by 16 points — ask your AI tool to vary correct answer placement in the next batch")
- Warning badge ("⚠ Imbalance Detected") when any letter exceeds 30% or falls below 20%

### Per-Book Breakdown
Table with columns: Book | A% | B% | C% | D% | Status (OK / Skewed)

This isolates which book's questions are causing the overall imbalance.

**Note:** The app already uses a seeded shuffle to randomize display order per question, so students never see the raw distribution. This panel exists for source-data hygiene and to guide AI prompting for future batches.

## 5. Enhanced Import Results Screen

After a CSV upload completes, the existing success screen is replaced with a quality summary.

### Summary Cards (3 across)
- **Low Distractor Score** — count of imported questions with `distractor_score < 60`
- **Possible Duplicates** — count of questions with >85% text similarity to an existing question in the bank (simple trigram or Levenshtein comparison, no AI needed)
- **Ready to Review** — count with no flags

### Distribution Bar (this batch)
Same A/B/C/D bar as the main panel but scoped to the just-imported batch. Includes the advisory tip for the next AI generation session if imbalanced.

### Flagged Questions Preview
First 3 flagged items shown inline with badge type (DISTRACTOR or DUPLICATE) and score. Link to see all flagged.

### Actions
- Import Another CSV
- Go to Review Queue → [N] pending (primary CTA)

## Architecture Notes

- Distractor quality check calls the Claude API server-side from the import route (`/api/import`). Use `claude-haiku-4-5-20251001` for cost efficiency — this is a classification task, not generation.
- Duplicate detection uses Levenshtein distance on `question_text` — no AI needed, runs synchronously on import.
- The review queue page is a server component that loads questions server-side; approve/flag actions are API calls (`PATCH /api/questions/[id]`).
- Answer distribution stats are a simple `GROUP BY correct_answer` SQL query — no caching needed at this scale.

---

## What This Does NOT Include

- Automated rewriting of weak distractors (admin rewrites manually or re-prompts the AI)
- Student-facing features (dashboards, study improvements, UX polish)
- Monetization or access control changes
- Full-text comparison against source books (books are not digitized; originality check is manual)
